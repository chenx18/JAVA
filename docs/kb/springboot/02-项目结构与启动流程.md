# 02-项目结构与启动流程

Spring Boot 项目有固定的基础结构。

看懂项目结构，才能知道代码应该放在哪里。

---

## 常见目录

```text
springboot-demo/
  pom.xml
  src/
    main/
      java/
        com/example/demo/
          SpringbootDemoApplication.java
      resources/
        application.properties
    test/
```

---

## pom.xml

`pom.xml` 是 Maven 项目的依赖配置文件。

它负责：

```text
声明项目依赖
声明 Java 版本
声明构建插件
管理第三方 jar 包
```

例如要使用 Spring Web，就会有：

```xml
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
</dependency>
```

---

## src/main/java

这里放 Java 业务代码。

常见类：

```text
启动类
Controller
Service
Repository / Mapper
Model / Entity
Exception
GlobalExceptionHandler
```

---

## src/main/resources

这里放资源和配置文件。

最常见：

```text
application.properties
application.yml
```

---

## 启动类

```java
@SpringBootApplication
public class SpringbootDemoApplication {
  public static void main(String[] args) {
    SpringApplication.run(SpringbootDemoApplication.class, args);
  }
}
```

`@SpringBootApplication` 表示这是 Spring Boot 项目入口。

`SpringApplication.run(...)` 会：

```text
启动 Spring 容器
扫描组件
加载配置
启动内嵌 Web 服务器
```

---

## application.properties

常见配置：

```properties
spring.application.name=springboot-demo
server.port=8081
```

后面还会配置：

```text
数据库连接
MyBatis 日志
接口文档
环境参数
```

---

## 一句话总结

```text
pom.xml 管依赖，src/main/java 放代码，src/main/resources 放配置，启动类负责启动整个项目。
```
