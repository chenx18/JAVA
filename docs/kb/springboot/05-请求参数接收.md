# 05-请求参数接收

Spring Boot Controller 用来接收 HTTP 请求。

不同位置的参数，要用不同注解接收。

---

## 一句话理解

```text
路径里的值用 @PathVariable
URL 问号后的查询条件用 @RequestParam
请求体 JSON 用 @RequestBody
请求头用 @RequestHeader
Cookie 用 @CookieValue
```

最常用的是前三个：

```text
@PathVariable
@RequestParam
@RequestBody
```

---

## @PathVariable：路径参数

`@PathVariable` 用来接收路径中的变量。

适合定位某个明确资源。

```java
@GetMapping("/departments/{id}")
public ApiResponse<DepartmentResponse> getDepartmentById(@PathVariable int id) {
  return ApiResponse.success(service.getDepartmentById(id));
}
```

请求：

```text
GET /departments/1
```

这里的 `1` 会进入 `id`。

常见场景：

```text
根据 ID 查详情
根据 ID 删除
根据 ID 修改
```

例如：

```text
GET /users/1
DELETE /roles/2
PUT /departments/update/3
```

---

## @RequestParam：查询参数

`@RequestParam` 用来接收 URL 问号后面的参数。

适合查询、筛选、分页、搜索。

```java
@GetMapping("/departments/search")
public ApiResponse<List<DepartmentResponse>> searchDepartmentsByName(
    @RequestParam String name) {
  return ApiResponse.success(service.searchDepartmentsByName(name));
}
```

请求：

```text
GET /departments/search?name=研
```

常见场景：

```text
模糊查询
条件筛选
分页参数
排序参数
状态筛选
日期范围
```

例如：

```text
GET /departments/list?pageNum=1&pageSize=10
GET /users?status=active
GET /orders?startDate=2026-01-01&endDate=2026-01-31
```

分页示例：

```java
@GetMapping("/departments/list")
public ApiResponse<List<DepartmentResponse>> getDepartmentList(
    @RequestParam int pageNum,
    @RequestParam int pageSize) {
  return ApiResponse.success(service.getDepartmentPages(pageNum, pageSize));
}
```

---

## @RequestParam 的 required 和 defaultValue

默认情况下，`@RequestParam` 是必填的。

如果请求不传，会报错。

可以设置非必填：

```java
@GetMapping("/departments/search")
public ApiResponse<List<DepartmentResponse>> search(
    @RequestParam(required = false) String name) {
  return ApiResponse.success(service.searchDepartmentsByName(name));
}
```

也可以设置默认值：

```java
@GetMapping("/departments/list")
public ApiResponse<List<DepartmentResponse>> list(
    @RequestParam(defaultValue = "1") int pageNum,
    @RequestParam(defaultValue = "10") int pageSize) {
  return ApiResponse.success(service.getDepartmentPages(pageNum, pageSize));
}
```

这样前端不传分页参数时，会默认查第一页、每页 10 条。

---

## @RequestBody：请求体 JSON

`@RequestBody` 用来接收请求体里的 JSON。

适合新增、修改、批量操作、复杂对象提交。

```java
@PostMapping("/departments/add")
public ApiResponse<DepartmentResponse> createDepartment(
    @RequestBody DepartmentCreateRequest request) {
  return ApiResponse.success(service.createDepartment(request));
}
```

请求体：

```json
{
  "id": 1,
  "name": "研发部"
}
```

Spring Boot 会把 JSON 转成 Java 对象：

```text
DepartmentCreateRequest
```

---

## @RequestBody + @Valid

如果请求对象字段上有校验注解，可以加 `@Valid` 触发校验。

```java
@PostMapping("/departments/add")
public ApiResponse<DepartmentResponse> createDepartment(
    @Valid @RequestBody DepartmentCreateRequest request) {
  return ApiResponse.success(service.createDepartment(request));
}
```

请求对象：

```java
@Data
public class DepartmentCreateRequest {

  @Min(value = 1, message = "部门ID必须大于0")
  private int id;

  @NotBlank(message = "部门名称不能为空")
  private String name;
}
```

如果参数不合法，会进入全局异常处理。

---

## @PathVariable + @RequestBody 组合

修改接口经常同时用路径参数和请求体。

```java
@PutMapping("/departments/update/{id}")
public ApiResponse<DepartmentResponse> updateDepartment(
    @PathVariable int id,
    @Valid @RequestBody DepartmentUpdateRequest request) {
  return ApiResponse.success(service.updateDepartment(id, request));
}
```

请求：

```text
PUT /departments/update/1
```

请求体：

```json
{
  "name": "研发中心"
}
```

这里：

```text
id 来自路径
name 来自请求体
```

---

## @RequestHeader：请求头参数

`@RequestHeader` 用来接收 HTTP 请求头。

```java
@GetMapping("/profile")
public ApiResponse<String> profile(@RequestHeader("Authorization") String token) {
  return ApiResponse.success(token);
}
```

请求头示例：

```text
Authorization: Bearer xxxxx
```

常见场景：

```text
Token
语言标识
客户端版本
设备信息
```

例如：

```text
Authorization
Accept-Language
User-Agent
```

---

## @CookieValue：Cookie 参数

`@CookieValue` 用来接收 Cookie。

```java
@GetMapping("/cookie")
public ApiResponse<String> cookie(@CookieValue("token") String token) {
  return ApiResponse.success(token);
}
```

现在前后端分离项目中，Token 更常放在请求头里。

所以 `@CookieValue` 先认识即可。

---

## MultipartFile：文件上传

文件上传常用 `MultipartFile`。

```java
@PostMapping("/upload")
public ApiResponse<String> upload(@RequestParam("file") MultipartFile file) {
  return ApiResponse.success(file.getOriginalFilename());
}
```

需要导入：

```java
import org.springframework.web.multipart.MultipartFile;
```

常见场景：

```text
上传头像
上传 Excel
上传附件
```

文件上传不是 JSON 请求体，通常使用 `multipart/form-data`。

---

## Map 接收多个查询参数

如果查询参数很多，也可以用 Map 接收。

```java
@GetMapping("/departments/query")
public ApiResponse<Map<String, String>> query(@RequestParam Map<String, String> params) {
  return ApiResponse.success(params);
}
```

请求：

```text
GET /departments/query?name=研&status=1
```

这种方式灵活，但类型不够明确。

学习阶段更推荐写明确参数或 Request DTO。

---

## 用对象接收查询参数

查询条件多时，也可以定义 QueryRequest。

```java
@Data
public class DepartmentQueryRequest {
  private String name;
  private Integer pageNum;
  private Integer pageSize;
}
```

Controller：

```java
@GetMapping("/departments/query")
public ApiResponse<List<DepartmentResponse>> query(DepartmentQueryRequest request) {
  return ApiResponse.success(service.queryDepartments(request));
}
```

请求：

```text
GET /departments/query?name=研&pageNum=1&pageSize=10
```

注意：GET 查询对象通常不加 `@RequestBody`。

Spring 会根据 query 参数自动绑定对象字段。

---

## 常见场景选择表

| 场景 | 推荐写法 |
|---|---|
| 根据 ID 查详情 | `@PathVariable` |
| 删除某个 ID | `@PathVariable` |
| 分页查询 | `@RequestParam` |
| 名称模糊搜索 | `@RequestParam` |
| 状态筛选 | `@RequestParam` |
| 新增对象 | `@RequestBody` |
| 修改对象 | `@PathVariable + @RequestBody` |
| 请求头 token | `@RequestHeader` |
| Cookie | `@CookieValue` |
| 文件上传 | `@RequestParam MultipartFile` |
| 多个查询条件 | `@RequestParam` 或 QueryRequest |

## 当前 Department 模块对应关系

```text
GET /departments/{id}
根据 ID 查询
=> @PathVariable
```

```text
GET /departments/search?name=研
根据 name 模糊查询
=> @RequestParam
```

```text
GET /departments/list?pageNum=1&pageSize=10
分页查询
=> @RequestParam
```

```text
POST /departments/add
新增部门 JSON
=> @RequestBody
```

```text
PUT /departments/update/{id}
路径里拿 id，body 里拿 name
=> @PathVariable + @RequestBody
```

## 常见错误

### 1. GET 请求用 @RequestBody 接收简单参数

不推荐：

```java
@GetMapping("/search")
public ApiResponse<?> search(@RequestBody String name) {
}
```

推荐：

```java
@GetMapping("/search")
public ApiResponse<?> search(@RequestParam String name) {
}
```

---

### 2. 路径里有变量，但没写 @PathVariable

路径：

```java
@GetMapping("/{id}")
```

参数应该写：

```java
@PathVariable int id
```

---

### 3. 查询参数用了 @PathVariable

如果请求是：

```text
/departments/search?name=研
```

就应该用：

```java
@RequestParam String name
```

不是：

```java
@PathVariable String name
```

---

## 一句话总结

```text
路径中定位具体资源，用 @PathVariable；URL 问号后的查询条件，用 @RequestParam；请求体 JSON 对象，用 @RequestBody。
```
