# 07-全局异常处理

全局异常处理用于把项目里的异常统一转换成接口返回。

---

## 为什么需要全局异常处理

如果没有统一处理，异常可能直接变成默认 500 页面或难看的错误栈。

实际项目希望返回统一格式：

```json
{
  "code": 400,
  "message": "name is blank",
  "data": null
}
```

---

## @RestControllerAdvice

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
}
```

作用：

```text
声明这是全局接口异常处理器
```

它会被 Spring 扫描并管理，不需要你在 Controller 里手动调用。

---

## @ExceptionHandler

```java
@ExceptionHandler(IllegalArgumentException.class)
public ApiResponse<String> handleIllegalArgument(IllegalArgumentException e) {
  return ApiResponse.fail(400, e.getMessage());
}
```

作用：

```text
指定某种异常交给这个方法处理
```

---

## 常见异常处理

```java
@ExceptionHandler(IllegalArgumentException.class)
public ApiResponse<String> handleIllegalArgument(IllegalArgumentException e) {
  return ApiResponse.fail(400, e.getMessage());
}

@ExceptionHandler(DepartmentNotFoundException.class)
public ApiResponse<String> handleDepartmentNotFound(DepartmentNotFoundException e) {
  return ApiResponse.fail(404, e.getMessage());
}

@ExceptionHandler(Exception.class)
public ApiResponse<String> handleOtherException(Exception e) {
  return ApiResponse.fail(500, "server error");
}
```

---

## 和 throw 的关系

Service 中抛异常：

```java
throw new DepartmentNotFoundException(id);
```

全局异常处理器接住异常：

```java
@ExceptionHandler(DepartmentNotFoundException.class)
```

最终返回统一 JSON。

---

## 一句话总结

```text
业务代码负责 throw，全局异常处理负责转换成 ApiResponse。
```
