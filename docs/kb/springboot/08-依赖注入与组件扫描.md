# 08-依赖注入与组件扫描

依赖注入和组件扫描是 Spring Boot 能自动组装项目的核心能力。

---

## 什么是依赖注入

例如：

```java
private final DepartmentService service;

public DepartmentController(DepartmentService service) {
  this.service = service;
}
```

这里不是你手动写：

```java
new DepartmentService(...)
```

而是 Spring 创建 `DepartmentController` 时，自动把 `DepartmentService` 传进来。

这叫依赖注入。

---

## 推荐构造器注入

当前阶段推荐这种写法：

```java
private final DepartmentService service;

public DepartmentController(DepartmentService service) {
  this.service = service;
}
```

优点：

```text
依赖明确
字段可以 final
更方便测试
不容易出现空指针
```

---

## 什么是组件扫描

Spring Boot 启动时会扫描组件注解。

常见组件注解：

```text
@RestController
@Service
@Repository
@Mapper
@RestControllerAdvice
```

被扫描到后，这些类会被 Spring 创建并管理。

---

## 扫描范围

`@SpringBootApplication` 默认扫描：

```text
启动类所在包
启动类所在包的子包
```

所以一般建议启动类放在根包：

```text
com.example.demo
```

业务类放在它下面：

```text
com.example.demo.controller
com.example.demo.service
com.example.demo.mapper
```

---

## GlobalExceptionHandler 为什么不用手动调用

因为它加了：

```java
@RestControllerAdvice
```

Spring 启动时会扫描并注册它。

当 Controller / Service 抛异常时，Spring 会自动把异常交给匹配的 `@ExceptionHandler` 方法。

---

## 一句话总结

```text
组件扫描负责发现类，依赖注入负责把对象传进去。
```
