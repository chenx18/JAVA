# 02-方法与构造函数重载

重载是 Java 中非常常见的写法。

它允许同一个类里出现多个同名方法或构造函数，只要参数列表不同。

---

## 一句话理解

```text
重载 = 同一个名字，支持不同传参方式。
```

比如：

```java
new DepartmentNotFoundException(1);
new DepartmentNotFoundException("研发部");
```

它们都在创建 `DepartmentNotFoundException`，只是传参不同。

---

## 构造函数重载

你可以在同一个类里写多个构造函数。

```java
public class DepartmentNotFoundException extends RuntimeException {

  public DepartmentNotFoundException(int id) {
    super("department not found by id: " + id);
  }

  public DepartmentNotFoundException(String name) {
    super("department not found by name: " + name);
  }
}
```

使用时：

```java
throw new DepartmentNotFoundException(1);
```

Java 会调用：

```java
DepartmentNotFoundException(int id)
```

使用：

```java
throw new DepartmentNotFoundException("研发部");
```

Java 会调用：

```java
DepartmentNotFoundException(String name)
```

---

## 方法重载

普通方法也可以重载。

```java
public class UserService {

  public User getUser(int id) {
    return null;
  }

  public User getUser(String name) {
    return null;
  }
}
```

调用：

```java
service.getUser(1);
service.getUser("tom");
```

Java 会根据参数类型选择对应方法。

---

## Java 如何判断是不是重载

Java 主要看参数列表。

参数列表包括：

```text
参数个数
参数类型
参数顺序
```

这些可以构成重载：

```java
public void test(int id) {}
public void test(String name) {}
public void test(int id, String name) {}
public void test(String name, int id) {}
```

---

## 参数名不同不算重载

下面这样不行：

```java
public void test(int id) {}
public void test(int departmentId) {}
```

虽然参数名不同，但参数列表都是：

```text
(int)
```

Java 会认为这是重复定义。

---

## 返回值不同不算重载

下面这样也不行：

```java
public int test(String name) {
  return 1;
}

public String test(String name) {
  return name;
}
```

因为参数列表一样：

```text
(String)
```

Java 不能只靠返回值判断调用哪个方法。

---

## 为什么要用重载

重载可以让代码更自然。

例如查找部门：

```java
findById(int id)
findByName(String name)
```

这两个方法名不同，也可以。

但异常类里常见写法是：

```java
new DepartmentNotFoundException(id)
new DepartmentNotFoundException(name)
```

因为它们表达的是同一种错误：

```text
部门不存在
```

只是定位方式不同。

---

## 重载和重写的区别

重载：

```text
overload
同一个类中
方法名相同
参数列表不同
```

重写：

```text
override
子类改写父类方法
方法名、参数通常相同
常见于继承和接口实现
```

你之前见过的：

```java
@Override
public String toString() {
  return "...";
}
```

这是重写，不是重载。

---

## 一句话总结

```text
重载看参数，重写看继承。
```
