# 05-异常基础

异常用于表达程序运行中出现了错误。

后端项目里，异常常用于中断错误流程，并交给全局异常处理统一返回。

---

## 异常体系最小理解

```text
Throwable
├─ Error
└─ Exception
   ├─ RuntimeException
   └─ 其他受检异常
```

当前阶段重点掌握：

```text
RuntimeException
IllegalArgumentException
自定义业务异常
```

---

## IllegalArgumentException

表示参数不合法。

常见场景：

```text
id <= 0
name 为空
email 格式不对
```

示例：

```java
if (id <= 0) {
  throw new IllegalArgumentException("id must be > 0");
}
```

---

## 自定义业务异常

业务异常用于表达业务失败。

例如：

```java
public class DepartmentNotFoundException extends RuntimeException {
  public DepartmentNotFoundException(int id) {
    super("department not found by id: " + id);
  }
}
```

它表示：

```text
参数本身可能合法，但业务数据不存在。
```

---

## System.out.println 和 throw 的区别

`System.out.println(...)` 只是打印信息。

它不会阻止代码继续执行。

`throw new Exception(...)` 表示真正抛出错误，当前流程会中断。

```text
打印是提示
throw 是错误流程控制
```

---

## 参数异常和业务异常

参数异常：

```text
输入本身不合法
例如 id <= 0
```

业务异常：

```text
输入格式合法，但业务上失败
例如 id = 99 查不到数据
```

---

## 一句话总结

```text
异常不是为了打印日志，而是为了表达错误并中断错误流程。
```
