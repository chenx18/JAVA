# 04-访问修饰符与封装

访问修饰符用于控制类、字段、方法能被谁访问。

封装是面向对象的重要思想：不把内部细节直接暴露出去。

---

## public

`public` 表示公开。

被 `public` 修饰的类或方法，可以被其他地方访问。

```java
public class DepartmentController {
}
```

```java
public Department getDepartmentById(int id) {
  return null;
}
```

Controller、Service 里的对外方法通常会写 `public`。

---

## private

`private` 表示私有。

只能在当前类内部访问。

```java
private int id;
private String name;
```

字段通常写成 `private`，避免外部随意修改。

---

## protected

`protected` 表示当前包和子类可访问。

初学阶段不常用，先知道即可。

---

## 默认访问权限

如果不写 `public`、`private`、`protected`，就是默认访问权限。

它表示同一个包内可访问。

学习阶段建议先明确写出访问修饰符，减少困惑。

---

## 封装怎么做

常见做法：

```java
public class Department {
  private int id;

  public int getId() {
    return id;
  }

  public void setId(int id) {
    this.id = id;
  }
}
```

字段私有，外部通过 getter / setter 访问。

---

## public 不是只用来定义类

`public` 可以修饰：

```text
类
构造函数
方法
```

`private` 可以修饰：

```text
字段
方法
构造函数
内部类
```

所以不要理解成：

```text
public 只定义类，private 只定义变量
```

它们本质上都是访问控制。

---

## 一句话总结

```text
public 让外部能用，private 把内部藏起来，封装让对象更安全、更可维护。
```
