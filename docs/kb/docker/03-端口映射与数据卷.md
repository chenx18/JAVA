# 03-端口映射与数据卷

运行数据库、Redis、Nginx 这类服务时，最常见的 Docker 配置是：

```text
端口映射
数据卷
```

---

## 1. 为什么需要端口映射

容器内部的服务运行在容器自己的网络环境中。

例如 MySQL 容器内部监听：

```text
3306
```

但本机 Spring Boot 想访问它，需要把容器端口映射到本机端口。

---

## 2. 端口映射写法

```yaml
ports:
  - "3307:3306"
```

含义：

```text
本机 3307 -> 容器内部 3306
```

Spring Boot 连接时写：

```properties
spring.datasource.url=jdbc:mysql://localhost:3307/week4_db
```

---

## 3. 为什么不用 3306:3306

如果本机已经安装了 MySQL，`3306` 可能已经被占用。

这时 Docker 启动会报：

```text
ports are not available
bind: Only one usage of each socket address is normally permitted
```

解决方式：

```yaml
ports:
  - "3307:3306"
```

容器内部还是 3306，本机访问改成 3307。

---

## 4. 什么是数据卷

容器可以删除，但数据库数据不能随便丢。

数据卷用于把容器内的数据保存到容器外部。

MySQL 常见配置：

```yaml
volumes:
  - week4-mysql-data:/var/lib/mysql
```

含义：

```text
把容器里的 /var/lib/mysql 数据目录保存到 Docker 数据卷 week4-mysql-data
```

---

## 5. 数据卷的作用

```text
容器删除后，数据仍可保留
重建容器后，可以继续使用原来的数据
避免数据库数据只存在容器内部
```

---

## 6. 端口和数据卷的区别

```text
ports：解决外部怎么访问容器服务
volumes：解决容器数据怎么持久保存
```

---

## 7. 一句话总结

```text
端口映射让本机访问容器，数据卷让容器数据不轻易丢失。
```
