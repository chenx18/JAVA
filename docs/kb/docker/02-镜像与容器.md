# 02-镜像与容器

Docker 里最重要的两个概念是：

```text
Image 镜像
Container 容器
```

理解了镜像和容器，就理解了 Docker 的基本运行方式。

---

## 1. 镜像是什么

镜像可以理解成：

```text
软件安装包 / 模板 / 快照
```

例如：

```text
mysql:8.4
redis:7
nginx:latest
```

镜像本身不是正在运行的软件，它只是一个可运行环境的模板。

---

## 2. 容器是什么

容器可以理解成：

```text
镜像运行起来后的实例
```

例如：

```text
mysql:8.4 是镜像
用 mysql:8.4 启动出来的 MySQL 服务就是容器
```

关系：

```text
镜像 = 模板
容器 = 运行中的实例
```

---

## 3. 一个镜像可以启动多个容器

同一个镜像可以创建多个容器。

例如：

```text
mysql:8.4 镜像
  -> mysql-dev 容器
  -> mysql-test 容器
```

它们来自同一个镜像，但可以有不同的容器名、端口、数据卷和配置。

---

## 4. 查看镜像

```bash
docker images
```

常见输出包含：

```text
REPOSITORY
TAG
IMAGE ID
CREATED
SIZE
```

例如：

```text
mysql  8.4
redis  7
```

---

## 5. 查看运行中的容器

```bash
docker ps
```

只显示正在运行的容器。

---

## 6. 查看所有容器

```bash
docker ps -a
```

包括已经停止的容器。

---

## 7. 容器状态

常见状态：

```text
Up：正在运行
Exited：已经停止
Created：已创建但未运行
Restarting：正在重启
```

---

## 8. 一句话总结

```text
镜像是静态模板，容器是镜像运行后的实例。
```
