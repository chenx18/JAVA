# 02-建库建表与字段类型

数据库里真正存数据的是表。

可以这样理解：

```text
数据库 = 一个文件柜
表 = 文件柜里的一个抽屉
字段 = 抽屉里的列
数据 = 每一行记录
```

---

## 1. 创建数据库

```sql
create database week4_db;
```

作用：创建一个名为 `week4_db` 的数据库。

如果使用 Docker Compose，可以通过 `MYSQL_DATABASE=week4_db` 在容器初始化时自动创建数据库。

所以通常只需要：

```sql
use week4_db;
```

---

## 2. 创建表

创建部门表：

```sql
create table department (
  id int primary key,
  name varchar(50) not null
);
```

含义：

```text
create table department：创建 department 表
id int primary key：id 是整数主键
name varchar(50) not null：name 是最多 50 字符的字符串，不能为空
```

---

## 3. int 类型

```sql
id int
```

表示整数。

常用于：

```text
id
年龄
数量
排序值
状态值
```

---

## 4. varchar 类型

```sql
name varchar(50)
```

表示字符串。

`50` 表示最大长度为 50 个字符。

常用于：

```text
名称
邮箱
手机号
标题
备注
```

---

## 5. primary key 主键

```sql
id int primary key
```

主键表示一条数据的唯一标识。

主键特点：

```text
唯一
不能为空
通常用来根据 id 查询、修改、删除
```

例如：

```text
id = 1 的部门
id = 2 的部门
```

不能有两条数据的 `id` 都是 1。

---

## 6. not null 非空

```sql
name varchar(50) not null
```

表示 `name` 字段不能为空。

如果插入数据时不传 `name`，数据库会拒绝。

---

## 7. 查看表结构

```sql
desc department;
```

作用：查看 `department` 表有哪些字段、字段类型和约束。

---

## 8. 当前必须记住

```sql
create table department (
  id int primary key,
  name varchar(50) not null
);

desc department;
```

一句话：

```text
建表就是定义这张表有哪些字段，每个字段是什么类型，有什么限制。
```
