# 05-主键约束索引基础

数据库不只是存数据，还要保证数据有规则、查询有效率。

本文先讲最基础的：

```text
主键
非空约束
唯一约束
索引
```

---

## 1. 什么是约束

约束就是数据库层面对数据的限制。

例如：

```text
id 不能为空
id 不能重复
name 不能为空
email 不能重复
```

这些规则不只可以在 Service 层校验，也可以在数据库层保证。

---

## 2. primary key 主键

```sql
id int primary key
```

主键表示一条数据的唯一标识。

特点：

```text
不能重复
不能为空
通常用来根据 id 查询、修改、删除
```

例如：

```sql
select * from department where id = 1;
update department set name = 'dev' where id = 1;
delete from department where id = 1;
```

这些都依赖 id 定位数据。

---

## 3. not null 非空约束

```sql
name varchar(50) not null
```

表示 name 不能为空。

如果插入：

```sql
insert into department (id) values (1);
```

数据库会拒绝，因为缺少 name。

---

## 4. unique 唯一约束

```sql
email varchar(100) unique
```

表示 email 不能重复。

常用于：

```text
用户名
手机号
邮箱
编码
```

主键和唯一约束区别：

```text
主键：一张表通常只有一个，用来标识整条数据
唯一约束：可以有多个，用来保证某些字段不重复
```

---

## 5. 什么是索引

索引可以理解成书的目录。

没有索引时，数据库可能要一行一行找。

有索引时，数据库可以更快定位数据。

例如：

```sql
select * from department where id = 1;
```

因为 `id` 是主键，MySQL 会自动给主键建索引，所以查询会更快。

---

## 6. 主键会自动创建索引

当你写：

```sql
id int primary key
```

MySQL 会自动为 `id` 创建索引。

所以按主键查询通常很快。

---

## 7. 创建普通索引

示例：

```sql
create index idx_department_name on department(name);
```

含义：

```text
给 department 表的 name 字段创建一个索引
索引名叫 idx_department_name
```

---

## 8. 索引不是越多越好

索引能提升查询速度，但也有代价。

缺点：

```text
占用额外空间
新增、修改、删除时需要维护索引
索引太多会影响写入性能
```

基础阶段可以先记住：

```text
经常作为查询条件的字段，才考虑建索引。
```

---

## 9. 当前必须记住

```text
primary key：主键，唯一且不能为空
not null：不能为空
unique：不能重复
index：索引，用来提高查询效率
```

一句话：

```text
约束保证数据正确，索引提高查询速度。
```
