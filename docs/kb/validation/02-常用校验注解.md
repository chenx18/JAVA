# 02-常用校验注解

Validation 的规则通常写在实体类字段上。

常用注解来自：

```java
jakarta.validation.constraints
```

---

## @NotNull

不能为 `null`。

```java
@NotNull(message = "ID不能为空")
private Integer id;
```

适合：

```text
Integer
Long
对象
集合
```

注意：`@NotNull` 只判断是不是 `null`，不判断字符串是不是空。

---

## @NotBlank

字符串不能为空。

```java
@NotBlank(message = "部门名称不能为空")
private String name;
```

它会拦住：

```text
null
""
"   "
```

适合校验字符串。

---

## @NotEmpty

不能为 `null`，长度也不能为 0。

```java
@NotEmpty(message = "列表不能为空")
private List<String> names;
```

常用于集合、数组、字符串。

和 `@NotBlank` 的区别：

```text
@NotBlank 只用于字符串，并且会判断空格
@NotEmpty 可用于集合，但不会把空格字符串当成空
```

---

## @Min 和 @Max

限制数字大小。

```java
@Min(value = 1, message = "ID必须大于0")
private Integer id;

@Max(value = 100, message = "数量不能超过100")
private Integer count;
```

---

## @Size

限制长度或集合数量。

```java
@Size(min = 2, max = 20, message = "名称长度必须在2到20之间")
private String name;
```

也可以用于集合：

```java
@Size(min = 1, message = "至少选择一个角色")
private List<Integer> roleIds;
```

---

## @Email

校验邮箱格式。

```java
@Email(message = "邮箱格式不正确")
private String email;
```

---

## Department 示例

```java
import jakarta.validation.constraints.Min;
import jakarta.validation.constraints.NotBlank;

public class Department {

  @Min(value = 1, message = "部门ID必须大于0")
  private Integer id;

  @NotBlank(message = "部门名称不能为空")
  private String name;
}
```

---

## 学习阶段先记哪些

优先掌握：

```text
@NotBlank：字符串不能为空
@NotNull：对象不能为空
@Min：数字最小值
@Size：长度限制
@Email：邮箱格式
```
