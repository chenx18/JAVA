# 04-全局处理校验异常

当 `@Valid` 校验失败时，Spring 通常会抛出 `MethodArgumentNotValidException`。

我们可以在 `GlobalExceptionHandler` 中统一处理它。

---

## 基础写法

```java
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

@RestControllerAdvice
public class GlobalExceptionHandler {

  @ExceptionHandler(MethodArgumentNotValidException.class)
  public ApiResponse<Void> handleValidationException(MethodArgumentNotValidException e) {
    String message = e.getBindingResult().getFieldError().getDefaultMessage();
    return ApiResponse.fail(400, message);
  }
}
```

---

## MethodArgumentNotValidException 是什么

一句话：

```text
MethodArgumentNotValidException = @Valid 校验请求体对象失败时，Spring 抛出的异常。
```

它通常发生在：

```java
public ApiResponse<Department> createDepartment(@Valid @RequestBody Department d)
```

如果 `Department` 字段不符合规则，就会进入这个异常处理方法。

---

## 取错误信息这句怎么理解

```java
e.getBindingResult().getFieldError().getDefaultMessage();
```

拆开理解：

```text
e：异常对象
getBindingResult()：获取参数校验结果
getFieldError()：获取其中一个字段错误
getDefaultMessage()：获取注解里写的 message
```

例如字段上写：

```java
@NotBlank(message = "部门名称不能为空")
private String name;
```

那么取出来的 message 就是：

```text
部门名称不能为空
```

---

## 防止空指针的写法

学习阶段可以先用基础写法。

实际项目中更稳一点可以这样写：

```java
@ExceptionHandler(MethodArgumentNotValidException.class)
public ApiResponse<Void> handleValidationException(MethodArgumentNotValidException e) {
  String message = "参数校验失败";

  if (e.getBindingResult().getFieldError() != null) {
    message = e.getBindingResult().getFieldError().getDefaultMessage();
  }

  return ApiResponse.fail(400, message);
}
```

---

## 是否需要背这串方法

不需要逐字背。

你只要知道：

```text
@Valid 校验失败
Spring 抛 MethodArgumentNotValidException
GlobalExceptionHandler 捕获
从异常里取出字段错误 message
返回统一 ApiResponse
```

这比硬背方法链更重要。
