# 03-参数绑定

参数绑定是 MyBatis 最重要的基础之一。

核心问题是：

```text
SQL 里的 #{id} 从哪里取值？
```

---

## 1. 单个普通参数

```java
@Select("select id, name from department where id = #{id}")
Department findById(int id);
```

调用：

```java
findById(1);
```

MyBatis 会把 `#{id}` 绑定为 `1`。

---

## 2. 单个对象参数

```java
@Insert("insert into department (id, name) values (#{id}, #{name})")
int insert(Department department);
```

这里：

```text
#{id}   -> department.getId()
#{name} -> department.getName()
```

参数名叫 `department`、`d`、`item` 都不重要。

重要的是对象里有对应属性和 getter。

---

## 3. 多个普通参数

如果方法有多个普通参数：

```java
int updateName(int id, String name);
```

建议使用 `@Param`：

```java
@Update("update department set name = #{name} where id = #{id}")
int updateName(@Param("id") int id, @Param("name") String name);
```

这样 SQL 里的：

```text
#{id}
#{name}
```

就能明确对应方法参数。

---

## 4. update 中为什么要 setId

接口常见写法：

```text
PUT /departments/2
Body: { "name": "dev" }
```

Controller 得到：

```text
@PathVariable int id = 2
@RequestBody Department department = { name: "dev" }
```

Body 里没有 id，所以对象里的 id 可能是默认值 0。

Service 需要：

```java
department.setId(id);
```

这样 Mapper 执行：

```java
@Update("update department set name = #{name} where id = #{id}")
```

才能从对象里取到正确的 id。

---

## 5. #{ } 和 ${ } 的区别

常用的是：

```text
#{参数名}
```

它会使用预编译参数，比较安全。

`${}` 是字符串拼接，容易有 SQL 注入风险。

入门阶段先记住：

```text
普通参数绑定优先使用 #{}
```

---

## 6. 一句话总结

```text
#{xxx} 会从 Mapper 方法参数中取值；对象参数会从 getter 中取属性值。
```
